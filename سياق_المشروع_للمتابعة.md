# 🗂️ وثيقة سياق مشروع ArabDiving — للمتابعة مع Claude

> انسخ هذه الوثيقة كاملة في بداية المحادثة على الحساب الآخر ليكمل Claude من حيث وقفنا.

---

## 0. كيف تبدأ مع Claude الجديد
قل له: «هذا سياق مشروعي، اقرأه وكمّل معي. المشروع في المجلد المربوط، والنشر عبر GitHub Desktop.» ثم الصق هذه الوثيقة.

---

## 1. نظرة عامة
- **المشروع:** ArabDiving (arabdiving.com) — مجتمع غوص عربي (RTL، عربي أولًا) موجّه للخليج (تركيز على السيدات السعوديات) لترويج الغوص في البحر الأحمر بمصر.
- **الطابع:** مجتمعي + حجز تجارب فاخرة وآمنة، مع خصوصية وملاءمة ثقافية.

## 2. التقنيات (Stack)
- **الواجهة:** Next.js 16 (App Router) + TypeScript، أنماط inline + `globals.css`، خط Cairo، اتجاه RTL. تُنشر على **Vercel**.
- **الخادم:** Express 5 + Mongoose، **MongoDB Atlas**، مصادقة JWT، bcryptjs، رفع صور/فيديو عبر **Cloudinary** (multer-storage-cloudinary). يُنشر على **Render** (خطة مجانية — لا Shell).
- **الدومين:** Namecheap (arabdiving.com) → DNS لـ Vercel، وCORS على Render.

## 3. سير النشر (مهم)
1. المستخدم يعمل **Commit + Push** من **GitHub Desktop**.
2. **Vercel** ينشر الواجهة تلقائيًا.
3. **أي تعديل في الـ backend** يتطلب **Manual Deploy** يدوي على Render (Deploy latest commit).

## 4. متغيرات البيئة
- **Render (backend):** `MONGO_URI`, `JWT_SECRET`, `CLOUDINARY_NAME`, `CLOUDINARY_KEY`, `CLOUDINARY_SECRET`, `CORS_ORIGIN` (قائمة مفصولة بفواصل، بدون / في النهاية)، `NODE_ENV=production`.
- **Vercel (frontend):** `NEXT_PUBLIC_API_URL` (رابط Render)، `NEXT_PUBLIC_SITE_URL` (https://arabdiving.com)، `NEXT_PUBLIC_GA_ID`.

## 5. بنية المجلدات
- `backend/src/` → `models/`, `controllers/`, `routes/`, `middleware/`, `config/`, `data/`, `server.js`, `seed.js`, `makeAdmin.js`.
- `frontend/app/` → الصفحات والمكوّنات و`lib/`.
- جذر المستودع: وثائق (`AUDIT.md`، `STRATEGY_الجمهور_والتسويق.md`، `SOP_اعتماد_مراكز_الغوص.md`، أدلة النشر).

## 6. ما تم بناؤه (الميزات)

### مجتمعي
- تسجيل/دخول (JWT)، أدوار member/admin.
- **منشورات المجتمع:** نص + صورة + **فيديو** (رابط YouTube/Vimeo أو رفع ملف)، تعديل، إعجابات، تعليقات قابلة للتعديل، مشاركة كل منشور برابطه `/community/[postId]` مع OG.
- **الأعضاء** `/members`: أصدقاء + مراسلة + خصوصية.
- **اللوج بوك**، الملف الشخصي + تعديله.

### نظام الأصدقاء والرسائل (للأصدقاء فقط)
- طلبات صداقة (إرسال/قبول/إزالة، قبول تلقائي متبادل).
- رسائل خاصة `/messages` + **شارة عدّاد غير المقروء** في القائمة.
- خصوصية: الصورة + البيانات (عام/أصدقاء فقط/مخفي) — تُفرض على الخادم.

### صفحات الفئات (قابلة للتعديل من CMS)
- الرئيسية، **الأطفال** (ألوان مرحة سماوي/أصفر)، **النساء** (وردي/بنفسجي)، **الشباب** (تركوازي/مرجاني)، الرحلات، **الدليل** (دليل الغوص)، **القصص** (+مسابقة شهرية)، **الباقات الفاخرة** `/retreats`.

### نظام المحتوى (CMS)
- `Content` model لكل صفحة (JSON) + `content-defaults.ts` + خطّاف `useContent`.
- محرّر أدمن عام `/admin/content` فيه `ImageField` و`VideoField`.
- **إظهار/إخفاء الصفحات** `/admin/pages` → `Settings.hiddenPages` → القائمة تُخفيها.

### المراكز الشريكة والحجز
- `PartnerCenter` بأوسمة الثقة السبعة + إدارة أدمن + بيانات تجريبية.
- `/family-booking`: بحث + فلاتر بالأوسمة + بطاقات.
- **معالج الحجز** `/family-booking/[id]` (٤ خطوات): التفاصيل ← الركاب (**نقطة تحوّل: لعبة الأطفال تنبثق**) ← الإضافات (**أسعارها قابلة للتعديل** من الأدمن) ← **التأكيد بلا دفع**: يختار العميل طريقة التواصل (واتساب/مكالمة/بريد) + أفضل وقت للاتصال، ثم تذكرة `AD-XXXXXX`.
- نماذج: `Booking` + `ChildProfile` (مقاسات الأطفال تُحفظ تلقائيًا). أدمن `/admin/bookings`.

### لعبة الأطفال «أبطال البحر الأحمر»
- مكوّن `SeaHeroesGame`: صنع شخصية ← غرفة معدات (تجمع المقاسات) ← عهد المحيط ← وسام. مستقلة `/game` ومدمجة في الحجز.

### المقاسات
- `SizeProfile` (شباب/شابات؛ للشابات إضافات: هودي + كاش مايوه). صفحات `/sizes/men` و`/sizes/women`، تظهر وتُعدّل من البروفايل، وأدمن `/admin/size-profiles`.

### التواصل
- **شات بوت عائم** يجمع (اسم/تواصل/رسالة) + زر واتساب. نموذج `ContactMessage`. أدمن `/admin/messages` (صندوق + **رقم واتساب قابل للتعديل** + تفعيل الشات).

### درجات الحرارة (Open-Meteo مجاني بلا مفتاح)
- `cityWeather.ts`: `AREAS` + `matchArea` (مع fallback) تغطي شرم، تيران، رأس محمد، دهب، نويبع/طابا، الغردقة، سفاجا، القصير، مرسى علم، أقصى الجنوب.
- حرارة هواء + ماء لحظية + منحنى سنوي SVG (`YearlyTempChart`) في صفحة كل موقع غوص + صفحة `/temperatures`.

### الملف الشخصي
- **تاريخ الميلاد** (مع حساب العمر)، نبذة (bio)، خصوصية، **تغيير كلمة المرور ذاتيًا** (بالتحقق من الحالية).
- الأدمن يعدّل أي مستخدم (بما فيها كلمة المرور) من `/admin/users`.

### عام
- **PWA** (manifest + أيقونات + service worker — قابل للتثبيت).
- GA4، صور OG، رؤوس أمان، تحديد معدّل، CORS allowlist.
- أزرار القائمة بطابع زر + ظل + **تمييز الصفحة الحالية بالذهبي**.
- لوحة الأدمن **متجاوبة على الموبايل** (قائمة هامبرجر).

## 9. ملاحظات تقنية مهمة لـ Claude الجديد
- **تحرير الملفات الكبيرة عبر bash heredoc** (الـ mount يتأخّر عن أدوات الملفات أحيانًا فيقتطع الملف). تحقّق دائمًا بـ `wc -l` + `tail` + `tsc`.
- **التحقق:** backend `node -e "require('./...')"`؛ frontend `rm -rf .next && npx tsc --noEmit` (يجب أن يكون exit 0).
- المسارات: مجلد العمل `C:\Users\Paysky\arabdiving`؛ في bash `/sessions/.../mnt/arabdiving`.
- بعد تعديل backend: ذكّر المستخدم بعمل **Manual Deploy على Render**.
