# 🚀 دليل نشر ArabDiving — مبسّط جدًا (زر بزر)

اتبع الخطوات بالترتيب. كل خطوة فيها بالضبط أي زر تضغط عليه.
كل المنصّات مجانية. ستحتاج بريدًا إلكترونيًا فقط.

> 💡 افتح كل منصّة في تبويب جديد، ولا تغلق التبويبات حتى تنتهي — ستنسخ روابط بينها.

---

## 🟢 الخطوة ١: ارفع المشروع على GitHub

1. ادخل على **github.com** → اضغط **Sign up** وأنشئ حسابًا (لو ما عندكش).
2. بعد الدخول، اضغط زر **+** أعلى يمين الصفحة → **New repository**.
3. في خانة **Repository name** اكتب: `arabdiving` → اضغط **Create repository**.
4. لرفع الملفات بدون أوامر: في صفحة المستودع اضغط **uploading an existing file**،
   ثم اسحب مجلّد المشروع كله (أو اضغط **choose your files**) → اضغط **Commit changes**.

✅ النتيجة: كل كودك صار على GitHub.

---

## 🟢 الخطوة ٢: قاعدة البيانات (MongoDB Atlas)

1. ادخل على **mongodb.com/atlas** → **Try Free** → سجّل حسابًا.
2. اختر الخطة المجانية **M0 (Free)** → اضغط **Create**.
3. من القائمة اليسرى → **Database Access** → **Add New Database User**:
   - اكتب اسم مستخدم وكلمة مرور (**احفظهما في ورقة**) → **Add User**.
4. من القائمة اليسرى → **Network Access** → **Add IP Address** → اضغط **Allow access from anywhere** → **Confirm**.
5. من القائمة اليسرى → **Database** → زر **Connect** → **Drivers** →
   انسخ الرابط الطويل الذي يبدأ بـ `mongodb+srv://...`
   - استبدل `<password>` بكلمة المرور التي حفظتها.
   - أضف `/arabdiving` قبل علامة الاستفهام `?`.

✅ احفظ هذا الرابط — اسمه **MONGO_URI**.

---

## 🟢 الخطوة ٣: حساب الصور (Cloudinary)

1. ادخل على **cloudinary.com** → **Sign up for free**.
2. من **Dashboard** ستجد 3 قيم — انسخها:
   - **Cloud Name**
   - **API Key**
   - **API Secret** (اضغط 👁 لإظهاره)

✅ احفظ الثلاثة.

---

## 🟢 الخطوة ٤: الباك‌إند (Render)

1. ادخل على **render.com** → **Get Started** → سجّل بحساب GitHub.
2. اضغط **New +** (أعلى) → **Web Service**.
3. اختر مستودع `arabdiving` → **Connect**.
4. اضبط الحقول التالية:
   - **Name:** `arabdiving-api`
   - **Root Directory:** `backend`
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Instance Type:** **Free**
5. انزل لقسم **Environment Variables** → اضغط **Add Environment Variable** لكل سطر:

   | المفتاح (Key) | القيمة (Value) |
   |---|---|
   | `MONGO_URI` | رابط Atlas من الخطوة ٢ |
   | `JWT_SECRET` | أي نص عشوائي طويل (اكتب 40 حرفًا عشوائيًا) |
   | `CLOUDINARY_NAME` | من الخطوة ٣ |
   | `CLOUDINARY_KEY` | من الخطوة ٣ |
   | `CLOUDINARY_SECRET` | من الخطوة ٣ |
   | `CORS_ORIGIN` | اكتب مؤقتًا `*` (نعدّله لاحقًا) |
   | `NODE_ENV` | `production` |

6. اضغط **Create Web Service** وانتظر حتى يكتمل (Live).
7. انسخ رابط الخدمة أعلى الصفحة (مثل `https://arabdiving-api.onrender.com`).

✅ افتح الرابط — لو ظهر `{"message":"ArabDiving API Running"}` فكل شيء سليم. هذا الرابط اسمه **API_URL**.

---

## 🟢 الخطوة ٥: الواجهة (Vercel)

1. ادخل على **vercel.com** → **Sign Up** بحساب GitHub.
2. اضغط **Add New...** → **Project** → اختر مستودع `arabdiving` → **Import**.
3. في **Root Directory** اضغط **Edit** واختر مجلّد **frontend**.
4. افتح **Environment Variables** وأضف:
   - **Key:** `NEXT_PUBLIC_API_URL`
   - **Value:** رابط Render (**API_URL** من الخطوة ٤)
5. اضغط **Deploy** وانتظر.
6. ستحصل على رابط مثل `https://arabdiving.vercel.app` — افتحه للتأكد أن الموقع يعمل.

---

## 🟢 الخطوة ٦: اربط دومين Namecheap

**في Vercel:**
1. افتح مشروعك → **Settings** → **Domains** → اكتب دومينك → **Add**.
2. Vercel سيعرض سجلّات DNS مطلوبة — اتركها مفتوحة لتنسخ منها.

**في Namecheap:**
3. **Domain List** → بجانب دومينك اضغط **Manage** → تبويب **Advanced DNS**.
4. احذف أي سجلّات قديمة (أيقونة سلة المهملات).
5. اضغط **Add New Record** وأضف ما عرضه Vercel، عادةً:
   - **A Record** · Host: `@` · Value: `76.76.21.21`
   - **CNAME Record** · Host: `www` · Value: `cname.vercel-dns.com`
6. احفظ (علامة ✔) وانتظر (من دقائق لساعات).

**رجوع إلى Render:**
7. **Environment** → عدّل `CORS_ORIGIN` ليصبح: `https://دومينك.com,https://www.دومينك.com`
8. اضغط **Save Changes** → **Manual Deploy** → **Deploy latest commit**.

✅ الآن موقعك يفتح على دومينك الخاص.

---

## 🟢 الخطوة ٧: تعبئة المواقع وإنشاء أول مشرف

أسهل طريقة من داخل Render:
1. افتح خدمة `arabdiving-api` → تبويب **Shell**.
2. اكتب: `npm run seed` ثم Enter (يرفع الـ41 موقع غوص).
3. سجّل حسابًا عاديًا من موقعك أولًا، ثم في الـ Shell اكتب:
   `npm run make-admin بريدك@الإيميل.com`

✅ سجّل دخولك بهذا الحساب → سيظهر زر **«لوحة الإدارة»** في الأعلى → ادخل `/admin`.

---

## ❓ لو حصلت مشكلة

- **الموقع يفتح لكن فارغ من المواقع:** نسيت `npm run seed`، أو `NEXT_PUBLIC_API_URL` غلط في Vercel.
- **رسالة CORS في الكونسول:** `CORS_ORIGIN` في Render لا يطابق دومينك بالضبط.
- **الصور لا ترفع:** راجع مفاتيح Cloudinary الثلاثة في Render.
- **الدومين لا يفتح:** انتظر انتشار DNS، وراجع سجلّات Advanced DNS.

> أي خطوة وقفت فيها، ابعتلي اسمها ورسالة الخطأ، وأقولك الحل بالضبط.
